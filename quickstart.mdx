## Overview

This guide will walk you through creating your first datalink and linking document properties to database values. We'll use the included employee datalink example to demonstrate the workflow.

<Note>
  Before starting, ensure you have completed the [installation](/installation) steps and have all three components running.
</Note>

## Prerequisites Checklist

Verify all services are running:

<Steps>
  <Step title="Alfresco Repository">
    Check that Alfresco is accessible at `http://localhost:8080/alfresco/`
    
    Default credentials: `admin` / `admin`
  </Step>
  
  <Step title="Database Mock Server">
    Verify the mock database is running on port 3005:
    
    ```bash
    curl http://localhost:3005
    # Should return: Venzia employee db mockup
    ```
  </Step>
  
  <Step title="Alfresco Content App">
    Access ACA at `http://localhost:4200` and log in with Alfresco credentials
  </Step>
</Steps>

## Your First Datalink

### Step 1: Verify Datalink Configuration

The employee datalink is already configured in the system. Let's verify it's loaded:

```bash
curl http://localhost:8080/alfresco/s/venzia/datalink/v1/datalinks
```

You should see a JSON response with available datalinks:

```json
[
  {
    "name": "employee",
    "title": "Company Employee",
    "aspectName": "dlnk:employee",
    "order": 10
  },
  {
    "name": "departments",
    "title": "Company Departments",
    "aspectName": "dlnk:departm",
    "order": 20
  }
]
```

<Accordion title="Understanding the Employee Datalink Configuration">
  The employee datalink is defined in `venzia-datalink/src/main/docker/datalink/datalink-employee.json`:

  ```json
  {
    "name": "employee",
    "title": "Company Employee",
    "description": "Example to datalink demo",
    "aspectName": "dlnk:employee",
    "aspectPropertyName": "dlnk:employee",
    "order": 10,
    "connectorRest": {
      "url": "http://localhost:3005/api/v1/public/employees/search",
      "authentication": {
        "type": "none"
      },
      "searchParam": "query"
    },
    "columns": [
      {
        "primaryKey": true,
        "name": "emp_no",
        "label": "Id",
        "type": "text",
        "hidden": true
      },
      {
        "name": "first_name",
        "label": "First Name",
        "type": "text"
      },
      {
        "name": "last_name",
        "label": "Last Name",
        "type": "text"
      },
      {
        "name": "hire_date",
        "label": "Hire Date",
        "type": "date",
        "format": "dd/MM/yyyy"
      }
    ]
  }
  ```

  **Key Points:**
  - Connects to a public endpoint (no authentication)
  - Maps to the `dlnk:employee` aspect
  - Displays employee name and hire date
  - Uses `emp_no` as the primary key (hidden from users)
</Accordion>

### Step 2: Upload a Test Document

In Alfresco Content App:

<Steps>
  <Step title="Navigate to a Folder">
    Go to **Personal Files** or any folder where you have write permissions
  </Step>
  
  <Step title="Upload a Document">
    Click the **Upload** button or drag and drop a file
    
    Any file type works—try a PDF, Word document, or text file
  </Step>
  
  <Step title="Verify Upload">
    Confirm the document appears in the file list
  </Step>
</Steps>

### Step 3: Link Employee Data to Document

Now let's link the document to an employee record:

<Steps>
  <Step title="Open Datalink Editor">
    Right-click on the uploaded document and select **Edit Datalink** from the context menu
    
    <Note>
      If you don't see the "Edit Datalink" option, verify that the ACA extension is properly loaded and that you're using the correct version of ACA.
    </Note>
  </Step>
  
  <Step title="Select Employee Datalink">
    In the datalink editor dialog:
    1. Choose **Company Employee** from the datalink type dropdown
    2. The form will load dynamically based on the datalink configuration
  </Step>
  
  <Step title="Search for an Employee">
    Start typing an employee name in the search field:
    - Try searching for "Adolfo"
    - The system queries the REST endpoint: 
      ```
      http://localhost:3005/api/v1/public/employees/search?query=Adolfo
      ```
    - Results appear in a searchable list showing:
      - First Name
      - Last Name
      - Hire Date
  </Step>
  
  <Step title="Select and Save">
    1. Click on an employee from the search results
    2. Review the selected employee information
    3. Click **Save** to apply the datalink to the document
    
    The document now has the `dlnk:employee` aspect applied with the linked employee data
  </Step>
</Steps>

### Step 4: Verify the Linked Data

Confirm that the datalink was applied successfully:

<Tabs>
  <Tab title="Via ACA UI">
    1. Select the document in ACA
    2. Open the **Info Drawer** (click the info icon)
    3. Expand the **Properties** section
    4. Look for the **Employee** aspect properties
    5. You should see the linked employee ID stored in `dlnk:employee`
  </Tab>
  
  <Tab title="Via REST API">
    Get the document properties via the Alfresco REST API:

    ```bash
    # Replace NODE_ID with your document's node ID
    curl -u admin:admin \
      "http://localhost:8080/alfresco/api/-default-/public/alfresco/versions/1/nodes/NODE_ID?include=properties"
    ```

    Look for the `dlnk:employee` property in the response:

    ```json
    {
      "entry": {
        "properties": {
          "dlnk:employee": "1",
          "cm:name": "my-document.pdf"
        }
      }
    }
    ```
  </Tab>
</Tabs>

## Try the Department Datalink

Now let's try the department datalink, which uses basic authentication:

<Steps>
  <Step title="Right-Click the Same Document">
    Open the datalink editor again by right-clicking on your test document
  </Step>
  
  <Step title="Select Department Datalink">
    Choose **Company Departments** from the dropdown
  </Step>
  
  <Step title="Search Departments">
    Search for a department:
    - Try "Recursos" or "Marketing"
    - This endpoint uses basic authentication (configured in the datalink JSON)
    - Credentials: `user2019` / `g3n3r4l`
  </Step>
  
  <Step title="Apply Department Link">
    Select a department and save
    
    The document now has both aspects:
    - `dlnk:employee`
    - `dlnk:departm`
  </Step>
</Steps>

<Note>
  A document can have multiple datalink aspects applied simultaneously, allowing you to link different types of data to the same document.
</Note>

## Create a Custom Datalink

Now let's create your own datalink configuration:

<Steps>
  <Step title="Create Configuration File">
    Create a new JSON file in the datalink configuration directory:

    ```bash
    cd venzia-datalink/src/main/docker/datalink
    ```

    Create `datalink-project.json`:

    ```json
    {
      "name": "project",
      "title": "Project Codes",
      "description": "Link documents to project codes",
      "aspectName": "dlnk:project",
      "aspectPropertyName": "dlnk:project",
      "order": 30,
      "connectorRest": {
        "url": "https://your-api.example.com/api/projects",
        "authentication": {
          "type": "basic",
          "username": "api_user",
          "password": "api_password"
        },
        "searchParam": "search"
      },
      "columns": [
        {
          "primaryKey": true,
          "name": "project_id",
          "label": "Project ID",
          "type": "text",
          "hidden": true
        },
        {
          "primaryKey": false,
          "name": "project_code",
          "label": "Code",
          "type": "text"
        },
        {
          "primaryKey": false,
          "name": "project_name",
          "label": "Project Name",
          "type": "text"
        },
        {
          "primaryKey": false,
          "name": "start_date",
          "label": "Start Date",
          "type": "date",
          "format": "dd/MM/yyyy"
        }
      ]
    }
    ```
  </Step>

  <Step title="Add Content Model Aspect">
    Edit the content model to add the new aspect:

    Open `venzia-datalink/src/main/resources/alfresco/module/aqua-datalink/model/datalink-model.xml`

    Add the new aspect:

    ```xml
    <aspect name="dlnk:project">
      <title>Project</title>
      <properties>
        <property name="dlnk:project">
          <type>d:text</type>
          <index enabled="true">
            <atomic>true</atomic>
            <stored>false</stored>
            <tokenised>false</tokenised>
          </index>
        </property>
      </properties>
    </aspect>
    ```
  </Step>

  <Step title="Rebuild and Restart">
    Rebuild the backend module and restart Alfresco:

    ```bash
    cd venzia-datalink
    ./run.sh reload_acs
    ```

    This will:
    1. Rebuild the ACS module
    2. Recreate the ACS Docker image
    3. Restart the ACS container with the new configuration
  </Step>

  <Step title="Verify New Datalink">
    Check that the new datalink is available:

    ```bash
    curl http://localhost:8080/alfresco/s/venzia/datalink/v1/datalinks
    ```

    You should see your new "project" datalink in the list.
  </Step>
</Steps>

### Required Response Format

Your REST API endpoint must return an array of objects:

```json
[
  {
    "field_name_1": "value1",
    "field_name_2": "value2",
    "date_field": "2024-01-15"
  },
  {
    "field_name_1": "value3",
    "field_name_2": "value4",
    "date_field": "2024-02-20"
  }
]
```

<Warning>
  The field names in your API response must exactly match the `name` values in your datalink column definitions.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Datalink Not Appearing in List">
    **Problem**: New datalink doesn't show up in the dropdown

    **Solutions**:
    1. Verify JSON syntax is valid
    2. Check the datalink file is in `src/main/docker/datalink/`
    3. Ensure the aspect exists in the content model
    4. Restart ACS: `./run.sh reload_acs`
    5. Check Alfresco logs for errors
    6. Verify via API:
       ```bash
       curl http://localhost:8080/alfresco/s/venzia/datalink/v1/datalinks
       ```
  </Accordion>

  <Accordion title="Can't Connect to REST Endpoint">
    **Problem**: Search returns no results or connection errors

    **Solutions**:
    1. Test endpoint directly with curl:
       ```bash
       curl "http://your-api.com/endpoint?query=test"
       ```
    2. Check authentication credentials
    3. Verify URL is accessible from Docker container
    4. For localhost endpoints, use host networking or `host.docker.internal`
    5. Check for CORS issues
    6. Review ACS logs for connection errors
  </Accordion>

  <Accordion title="Wrong Data Displayed">
    **Problem**: Datalink shows incorrect or missing information

    **Solutions**:
    1. Verify API response format matches column definitions
    2. Check that field names are exactly the same
    3. Ensure primaryKey is set on one column
    4. Test API response:
       ```bash
       curl "http://localhost:3005/api/v1/public/employees/search" | jq
       ```
    5. Verify column types match data types (text, date, etc.)
  </Accordion>

  <Accordion title="Authentication Fails">
    **Problem**: 401 Unauthorized errors

    **Solutions**:
    1. Verify credentials in datalink JSON
    2. Test authentication manually:
       ```bash
       curl -u username:password http://your-api.com/endpoint
       ```
    3. Check authentication type is correct ("basic", "none")
    4. For the mock database, use `user2019` / `g3n3r4l`
    5. Review API server logs for auth errors
  </Accordion>
</AccordionGroup>
